I joined FPT Software in late 2016 as a manual tester. By the time I left in
2020 I was writing code for a living, but the tester DNA never quite left the
building. This is a note about why I'm grateful for that, and how it still
shapes the way I open a pull request today.

## The first instinct is still "what breaks this"

Even now, the first thing my brain does when I read a spec isn't "how do I
build it" — it's "what's the input that breaks it." Empty string. Zero.
Negative. A user who refreshes mid-submit. A network that hangs at 99%.
